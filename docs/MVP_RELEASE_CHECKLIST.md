# MVP release checklist

## Before sharing with testers

- [ ] PR-006 merged.
- [ ] PR-007 merged.
- [ ] Test URL or local test path prepared.
- [ ] Tester list prepared with 5 to 10 people.
- [ ] At least 2 non-technical homeowners included.
- [ ] At least 1 construction or roofing person included.
- [ ] At least 1 ASTRA/internal reviewer included.

## Pricing

- [ ] Active prices confirmed for MVP testing.
- [ ] Non-binding disclaimer visible.
- [ ] Result clearly says informative estimate.
- [ ] Result does not present a final offer.
- [ ] Example estimates reviewed by ASTRA before sharing widely.

## Legal / GDPR

- [ ] Consent required before inquiry.
- [ ] Privacy/legal text marked for final review before public advertising.
- [ ] No production customer data used in testing.
- [ ] No paid ads until test feedback is reviewed.

## Technical

- [ ] `npm test` passes.
- [ ] Static app loads.
- [ ] `mailto:` tested.
- [ ] Mobile width checked.
- [ ] Desktop checked.
- [ ] App works without backend.

## UX

- [ ] Main form labels are understandable.
- [ ] Non-binding estimate message is visible.
- [ ] Required fields are clear.
- [ ] Error messages are understandable.
- [ ] Result summary is readable on mobile and desktop.

## Lead handling

- [ ] Example inquiry email reviewed by ASTRA.
- [ ] ASTRA can understand the email content.
- [ ] Internal reviewer knows how to receive and evaluate test inquiries.
- [ ] Process exists for collecting tester feedback outside the app.

## After test

- [ ] Collect feedback from all testers.
- [ ] Identify confusing fields.
- [ ] Review whether `mailto:` is reliable enough.
- [ ] Review whether estimate ranges feel credible.
- [ ] Decide next PR: mobile/visual polish, backend/lead delivery or pricing refinement.
