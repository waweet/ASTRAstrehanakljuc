# PR-001 - Stabilize initial ASTRA roof quote MVP

## Role

You are the execution coding agent for the ASTRA Streha na ključ webapp.

The strategic orchestrator is Tetti. Follow `AGENTS.md` strictly.

## Current verified project state

The repository currently contains:

- static frontend prototype in `app/`,
- pricing engine in `src/pricing-engine.js`,
- pricing config in `src/pricing-config.js`,
- tests in `tests/pricing-engine.test.js`,
- product docs in `docs/`,
- project rules in `AGENTS.md`.

## Goal

Improve the initial MVP so it is ready as a clean first GitHub PR baseline for further development.

This PR should make the current static prototype more robust, readable, test-backed and easier to continue.

## Scope

Do all of the following:

1. Inspect the whole repository.
2. Run the existing tests.
3. Verify that the static app runs locally.
4. Improve code quality where needed, but keep the current no-dependency static architecture.
5. Add defensive input handling to the calculator.
6. Improve the lead form so generated email body includes the latest calculated estimate if available.
7. Add at least two additional pricing tests:
   - invalid unknown covering should throw a clear error,
   - complex roof with very difficult access should produce a higher range than a simple roof with the same area and covering.
8. Add a short `docs/NEXT_STEPS.md` with a proposed PR sequence.

## Non-goals

Do not:

- migrate to Next.js, React, Vue, Svelte or any other framework,
- add a backend,
- add a database,
- add real email sending,
- add analytics,
- add authentication,
- invent final ASTRA prices,
- change the product promise from informative estimate to final offer,
- add external dependencies unless absolutely necessary.

## Acceptance criteria

The PR is acceptable only if:

- `npm test` passes,
- the app can still run with `npm start`,
- calculation is still isolated in `src/pricing-engine.js`,
- pricing numbers remain in `src/pricing-config.js`,
- lead email includes contact data, description and latest estimate,
- UI still clearly says the price is informative,
- `docs/NEXT_STEPS.md` exists,
- no unrelated files are changed.

## Evidence required

In your final report include:

- exact commands run,
- test output summary,
- files changed,
- what was manually smoke-tested,
- any known limitations,
- suggested next PR.
